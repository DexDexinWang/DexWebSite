Redux

Redux Development History :

MVC (Model, View, Controller): Request -> Controller -> Model -> View (One Direction). However, in some real projects, it always allows View accesses Model. Finally, the whole projects become more and more complexity.

Flux: Action -> Dispatcher -> Store -> View - Action -> Dispatcher (Cycle).

Compare MVC and Flux

  1. MVC shows Controller to requester. Flux shows Dispatcher to requester.
  2. For adding a new feature, MVC need to create a new controller, and Flux will only add one Action.
