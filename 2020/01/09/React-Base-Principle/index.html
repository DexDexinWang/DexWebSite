React

React is a Javascript library for building user interfaces. It defines a new idea(Reactive Programming) to create UI, UI = render (data).

HTML DOM Tree (Document Object Model) is constructed as a tree of Objects for HTML. Brower will refresh whole DOM tree if there is any change. Hence, Web developers prefer reducing DOM manipulation to save cost.

Virtual DOM is a representation of a UI is kept in memory and synced with real HTML DOM tree. It will notify which DOM is changed to reduce cost for page change.

High Cohesion: React allows css, js, and html are in the same js file.

Low Coupling: React define components in a easy way with standard interface.

React Refresh Rule: Any change in “prop” and “state” will cause Virtual DOM Refresh.

Component Lifcycle: Mounting (first render in DOM tree), Updating (refresh), Unmounting(Delete from DOM tree).

Lifecycle Updated on 01/09/2020

Mounting: constructor() –> static getDerivedStateFromProps() –> render() –> componentDidMount()

Updating: static getDerivedStateFromProps() –> shouldComponentUpdate() –> render() –> getSnapshotBeforeUpdate() –> componentDidUpdate()

unmounting: componentWillUnmount()

Tips

  • JSX (JavaScript eXtension) allows developers use defined components as the same as using HTML element in .js files. The different between React component and HTML element is checking whether the first character is capitalized or not. Also, JSX will define render scope to improve efficency.

  • pckage.json defines some packages that will be used in current project. Runing “npm start” means running “react-scripts start” in npm package which is added by “create-react-app”.

  • Every react page, class, or component should import React because JSX will translate content as React expressions.

  • prop is used to pass parameters from outside to inside. It looks like setup attributes to HTML element. However, HTML only passes string value for each attribute. In React, it could pass any JavaScript data type.

  • Render() should only return one element. For more return components, it could be covered by div.

  • If any component want to define the constructor(props), it should write super(props) in the first line. Otherwise, the instances of this component could not get props.

  • propTypes is used to define prop standards. However, this feature will cause more cost and it is useful only for developers. Suggestion, it could be used in testing period. Then, this part could be deleted it before deploying the program.

  • defaultProps makes better program.

  • access state in correct way to refresh page.

  • Showing Component priority with ‘caption’ parameter in props.

  • Context feature is supplied by React for accessing content in whole DOM tree. It means every node in DOM Tree could access the some context without passing props. Normally, it is used for accessing store in Redux.
