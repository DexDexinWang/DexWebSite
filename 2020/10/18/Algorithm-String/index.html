Substring is a sequence of consecutive characters in a string.

Subsequence is any sequence of character in a string in their original order.

All substrings are subsequences, but the converse is not ture. A substring or subsequence is claled proper if it is not the entire string.

Tire Structure

A trie is a rooted tree that maintians a set of strings.
Each string in the set is stored as a character chain that starts at the root node. If 2 strings have a common prefix, they also have a common chain in the tree.

Dynamic Programming

Longest Common Subsequence: The Longest Common Subsequence of two strings is the longest string that appears as a sbusequence in both strings.

E.g. TOUR and OPERA is OR

Edit Distance: The edit distance between two strings denotes the minimum number of editing operations that transform the first string into the second string.

E.g. TOUR and OPERA is 2 With Delet, Update, Insert

String Hashing

String Hashing is a mthoed to efficiently check whether 2 strings are qeual by comparing their hash values.

Hash Value is an integer that is calculated from the characters of the string.

Polynomial Hashing is a popular way to implement string hashing

In addition, some questions will ask to compare multiple substings.

The general way is to create 2 arrays for preprocessing within O(n) times, and each compare will cost O(1) time complexity.

Solved Problems with O(n) preprocess and O(1) each compare:

  • Pattern Matching (e.g. ABCABABCA, ABC ==> 2)
  • Distinct Substrings (elg. ABABAB, distinct substring of length 3 ==> ABA BAB)

Hash Value Risk:

An evident risk when comparing hash values is a collision which means that two strings have different contents but equal hash values.

Collisions are always possible because the number of different strings is larger then the number of different hash values. However, the probability of a collision is small if the contents X and Y are carefully chosen.

A usual way is to choose random constants near 10 ^ 9.
