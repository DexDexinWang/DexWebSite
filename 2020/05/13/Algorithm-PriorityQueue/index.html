Graph Tree and ListNode

The defination in Wiki is that “A tree structure or tree diagram is a way of representing the hierarchical nature of a structure in a graphical form”. In another wrod, the tree structure has no loop to represents each node in hierarchical level. Also, ListNode structuer is another kind graphical form.

Binary Tree is kind of tree structure in which node has at most 2 nodes representing left node and right node.

Tree Traverse

Normally, there are 3 ways to traverse each node in a tree.

  • Pre-order: root -> left -> right
  • In-order: left -> root -> right
  • Post-order: left -> right -> root

If the depths is too large, it will post “stack overflow“ exception. Hence, we could use iteration and stack to do Pre-order, In-order, and Post-order.

Balanced binary tree: is commonly defined as a binary tree in which the depth of the left and right subtrees of every node differ by 1 or less.
If a tree has n number of nodes, and it is balanced, then the height(level) of the tree = O(log_2(n)).

Conclusion 1: if a tree has a number of nodes, and it is balanced, then the height(level) of the tree = O(log_2(n)).

Complete binary tree: is a binary tree is which every level, except possible the last, is completely filled, and all nodes are as far left as possible.

Conclusion 2: If a tree is a complete tree, then it must be a balanced tree.

Binary Search Tree: for every single node in the tree, the values in its left subtree are all smaller than its value, and the value in its right subtree are all larger than its value. Cannot have duplicated node. If yes, a key-value pair could be used to replace key.
Conclusion 3: If we print the value of the nodes in BST in in-order sequence, then it must from an ascending order.

Time complexity:

  • Search: worst case O(n) ; average case O(logn)
  • Insert: worst case O(n) ; average case O(logn)
  • Delete: worst case O(n) ; average case O(logn)

The worst case is casued by un-balance. The advantage of binary search tree is AVL tree or Red-Black tree which will maintain a balance tree.

Red-Black Tree

It maintains 3 points:

  • Every node has a color either red or black.
  • Root of tree is always black.
  • There are no two adjacent red nodes (A red node cannot have a red parent or red child).

The implementation could is more complicated. Refer to Princeton class.

Binary Tree Algorithm Logic

Binary Tree is the most popular tree structure to be used for answering algorithm problems in interviews. Typically, most solution use recusion to traverse trees to return result. The solution will will consider 3 points:

Tree + Recursion: Space complexity is O(height) like DFS where hight may equal n.

  • What do you expect from your left child / right child?
  • What do you want to do in the current level?
  • What do you want to report to your parent?

E.g. Maximum Depth of Binary Tree

However, some kind questions will ask result by checking nodes level by level such as 993. Cousins in Binary Tree and 107. Binary Tree Level Order Traversal II. Then, the solution will consider iteration with a queue to do BFS. Space complexity is O(2^height) like BFS.
