Object-Oriented Design

Paking System Design Steps:

Step 1

Understand/Analyze the USE CASE.

Use case: Describe the parking lot building? Vehicle monitoring? What kind of parking lot?
Some other questions that may affect your design:

  • One level or multiple levels?
  • Parking – Spot / Vehicle sizes?
  • Track the location of each vehicle?

Step 2

Design Models (class) and relationships between Models or class

Single responsibility Principle: each class should have only one job but (mutiple APIs)

  • Vehicle
  • Parking Spot
  • Level
  • ParkingLot - Time, Price attribute.

Class relationships:

Association: a general binary relationship that describes an activity between two classes. 1-1

Aggregation/Composition: a special form of association, which represents an ownership relationship between two classes. (has-a).

Inheritance:

Vehicle – Parking spot, Association

Level – Parking Spot, Composition

Vehicle – Car, Truck

Step 3

For complicated designs, first focus on public methods (APIs): how to revoke your APIs?
Functionality:

  • Base functionality: for a given vehicle, tell whether there is available spot in the parking lot.
  • Possible extensions: provide available spot locations, assign spot to the vehicle.
    …… Generate Classes and APIs with comments ……

Step 4

Complete all implementation.

Assumption: multiple levels and check vehicle size.

  1. APIs:

    Boolean hasSpot (Vehicle v);

    Boolean park (Vehicle v);

    Boolean leave (Vehicle v);

  2. Classes:
    ParkingLog, Level, Vehicle, ParkingSpot, Car, Truck

OOD: Performance/Efficiency is still important! This is where you use data structures and algorithms!

ENUM

It is a special data type that enables for a variable to be a set of predefined constants.

  1. The enum declaration defines a class. The enum class body can include methods and other fields.
  2. The compiler automatically adds some special methods when it creates an enum. Such as values() method. ( for(Day day : Day.values()) {} );
  3. The constructor for an enum type must be package-private or private access. It automatically creates the constants that are defined at the beginning of the enum body. You cannot invoke an enum constructor yourself.
Design Pattern

  1. Builder

  2. Factory

  3. Singleton
