JAVA STRING CLASS

Sting is a very special class in Java Programming language. String and all the wrapper class objects are immutable class that means if an string is created, it cannot be chagned.

String is not Primitive Type. String is extended from Object which has many methods such as “indexOf”

String Creation

Whenever Java encounters a string literal in your code, the compiler creates a String Object with its value. If there are mutiple string objects with the same value. It means these string objects refer to the same object that Java Created.

Conclution:

There is no need for maintaining several copies of String objects with the same iteral, since String objects are immutable.

Usually compiler and JVM will optimize for the number of String objects created, it will maintain an intern area in Heap, for the same String literal it will only has one String object associated.

String Comparation

Comparing two strings lexicographically. If two strings are different, then either they have different characters at some index that is a valid index for both strings, or their lengths are different, or both.

String Method

Any method provied by String class will always returns a new string without modifiying string objects.

StringBuilder and StringBUffer:

They are intermediate classes that are modifiable since String is Immutable.
Different between StringBuilder and StringBuffer
• They are doing exactly the same thing
• StringBuffer is “thread-safe”, each of the methods are synchronized.
• StringBuilder is not thread-safe, it is intended to be used in single thread environment.
