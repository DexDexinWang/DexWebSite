JAVA Object-Oriented Paradigm: Everything in the world is an object

  1. Each object has a type
  2. Each object is defined by two kinds of information

    2.1. Status: the “field”s – what things it maintains

    2.2. Behavior: the “method”s (functions) – what it can do

JAVA Main Method: Main Method is the entry point of Java programs. Each project has only one Public Main Class. The name is fixed as “main” which is called by the JVM as entry point for an application.

e.g. public static void main(String[] args)]

JAVA Method:

E.g. public static Integer methodName(int a, String b)

  • Modifier (public static)
  • Return type (Integer)
  • Name of the method (methodName)
  • List of parameters (int a, String g)

JAVA Method (Function) Operation: If you run a method, the method will evaluate all parameters. For the method Evaluation, it requires all parameters should be certain values.


E.g. Return methodA(array, methodB(array)).

It should operate methodB(array) first because it should be calculated before running methodA.

JAVA evalation orders is from left to right.

Reference & Dereference:

E.g.
{List} {list} {=} {new LinkedList();}
{Declaration} {Reference} {Instantiation} {Instantiation}

  • Declaration: associate a variable name with an object type
  • Reference: list is a reference which is a kind pointer to point data.
  • Instantiation: After “new” an object, the “=” will return its reference back to a “variable” (reference).
  • Initialization: The new operator is followed by a call to a constructor, which initializes the new object.

E.g.list.get(1) = 1314521

Dereference: the number ‘1314521’ will be assign to data which list points to.

Java-JVM Memory: Stack and Heap:

Stack: LIFO, a call stack is a stack data structure that stores information about the active subroutines of a computer program. This kind of stack is also known as an execution stack, program stack, control stack, run-time stack, or machine stack, and is often shortened to just “the stack”. It is used to track the instructions.

Heap: Java objects reside in an area called the heap. The heap is created when the JVM (Java Virtual Machine) starts up and may increase or decrease in size while the application runs.

Heap & Stack Comparison:

  • Local primitive type variables are always on stack
  • Whenever an object is created, its always stored in the Heap space and stack memory contains the reference to it.
  • Memory management in stack is done in LIFO manner; memory management in heap is more complex and requires complex algorithm
  • Stack is like the data structure “stack” because both are LIFO, heap has little to do with the data structure “heap”.
  • Stack variable is destroyed after out of scope. Heap variable is destroyed by the garbage collector.
  • (advanced & optional) Each thread has its own stack; multiple threads share the same heap.
  • (advanced & optional) When stack memory is full, Java runtime throws java.lang.StackOverFlowError
    Whereas if heap memory is full, it throws java.lang.OutOfMemoryError: Java Heap Space error.
  • (Advanced & optional) Stack is smaller than heap; storing data to stack is faster than heap.
  • Frequently, Stack is far smaller than Heap. It is easier to have OVER FLOW ERROR.

Not all variables saved in Stack, but all objects are saved in Heap.

Primitive Types (save it in stack): Defined with small capital letter in name.

Wrapper class (save object in heap): Defined with big captial letter in name. is just a wrapper of the corresponding primitive type, the Object representation of primitive type values.

  • boolean <-> Boolean
  • byte <-> Byte
  • char <-> Char
  • short <-> Short
  • int <-> Int
  • long <-> Long
  • float <-> Float
  • double <-> Double

This is string literal. When you declare string like this, you are actually calling intern() method on St ring. This method references inernal pool of string objects. If there already exists a string value “This is string literal”, then str will reference of that string and no new String object will be created. This is string object. In this method JVM is forced to create a new string reference, even if “this is string created by new operator” is in the reference pool.

Wrap object is similar to String, all the wrapper class objects are IMMUTABLE – internal values can not be changed after initialization.

Hints:

  • Generic type can not be primitive type, List, there can not be List.
  • It can help provide useful functionalities and contracts (json serializer: obj -> json)
  • How do you represent a “not existed” int value? “null” –> Using Wrapper class object.

Autoboxing && Unboxing

Autoboxing is the automatic conversion that the Java compiler makes between the primitive types and their corresponding object wrapper classes.

JAVA Parameter Passing:
When a method is called, how does it actually pass parameters?
Java only has no conception about Pass-By-Reference, so Java aways use Pass-By-Value (copy).

  • Primitive type: copy the value itself
  • Objects: copy of the object reference

Static Keyword

Any Static members (fields, methods, classes) belong to class, not object. The member could be accessed by [ClassName].[Static field].

Java does not know how to create an instance, so it is using Main function, “public static void main(String[] args)”.

Advantage, Nested Static Class:

  • The nested static class could be instantiated before the instantiated outer class.
  • To solve dynamic constructor problem, the nested static class, “XXBuilder” could invoke the private outer class constructor.

Final Keyword

Constant. “Once assigned, cannot be changed.”

  • Final Class: A class that cannot be derived. It has no child (no inheritance).
  • Final method: A method has cannot be overridden. It could not be override.
  • Final field: A variable that once assigned, it cannot be assign again even in constructor. It could not be changed;

Other Keywords:

Classic Types (objects): stored in heap, operated by references (reference object).

‘this’ Keyword: The current(self) instance. Mandatory if argument name is the same as field name.

‘null’ keyword: is special value for the reference. It means that reference is empty.

NullPointerException: It means it makes JAVA dereferences the reference (NULL).
