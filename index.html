Linked List

GeeksforGeeks: A linked list is a linear data structure, in which the elements are not stored at contiguous memory locations. The elements in a linked list are linked using pointers.

In simple words, a linked list consists of nodes where each node contains a data field and a reference(link) to the next node in the list.

Single & Double Linked List Defination

By personal experience, the Single Linked List is used to answer algorithm questions. The double Linked List is used to generate new structure.

Algorithm Problem

For most Linked List Algorithm problem, there are some tips will be helpful to solve the problem.

  1. The HEAD node should be saved util final output.
  2. Check NPE first before using next pointer and value.
  3. At Beginning, check corner cases will be helpful to make solution better and save time.
  4. Define an extra node to handle head will make clear logic.

Reverse Linked List

  1. Defining 2 pointers {walker = walker.next and runner = runner.next.next} could find middle point which will be use multiple algorithm problems.

    5.1. While (runner != null && runner.next != null) will make the walker to stop at ( (n/2+1), from 1 to n).

    5.2. While (runner.next != null && runner.next.next != null) will make the walker to stop at ( (n+1)/2 , from 1 to n).

For example:

5.1.

[1 2 3 4 5] ===> walker is 3; runner is 5

[1 2 3 4 5 6] ===> walker is 4; runner is null

5.2.

[1 2 3 4 5] ===> walker is 3; runner is 5

[1 2 3 4 5 6] ===> walker is 3; runner is 5

Inheritance, Interface, Abstract Class:

Inheritance:

Definitions: A class that is derived from another class is called a subclass (also a derived class, extended class, or child class). The class from which subclass is derived is called a superclass (also a base class or parent class).

Abstract:

Definitions: A class must be declared abstract when it has one or more abstract methods. A method is declared abstract when it has a method heading, but no body – which means that an abstract method has no implementation code inside like normal methods do.

Interface:

An interface differs from an abstract class has only abstract methods.

Difference between Abstract and Interface in Java:

  1. Java does not allow multiple inheritance. In Java, a class can only derive from one class, whether its abstract or not. However, a class can implement multiple interfaces.

  2. An abstract class may provide some methods with definitions – so an abstract class can have non-abstract methods with actual implementation details. An abstract class can also have constructors and instance variables as well. An interface, however, can not provide any method definitions – it can only provide method headings. Any class that implements the interface is responsible for providing the method definition/ implementation.

  3. Abstract: is a something. Interface: has a function.

Override: a subclass o child class to provide a specific implementation of a method that is already provided by one of its superclasses or parent classes.

Polymorphism: A call to a member function will cause a different function to be executed depending on the type of object that invokes the function.

Override VS. Overload
Override is used when you redefine a method that has been defined in a parent class (using the same signature). Resolved at running time.
Overload is used when you define two methods with the same name, in the same class, distinguished by their signatures (different), Resolved at compile time.

STAR Method

The STAR method is a structured manner of responding to a behavioral-based interview question by discussing the specific situation, task, action, and result of the situation you are describing.

Situation: Describe the situation that you were in or the task that you needed to accomplish. You must describe a specific event or situation, not a generalized description of what you have done in the past. Be sure to give enough detail for the interviewer to understand. This situation can be from a previous job, from a volunteer experience, or any relevant event.

Task: What goal were you working toward?

Action: Describe the actions you took to address the situation with an appropriate amount of detail and keep the focus on YOU. What specific steps did you take and what was your particular contribution? Be careful that you don’t describe what the team or group did when talking about a project, but what you actually did. Use the word “I,” not “we” when describing actions.

Result: Describe the outcome of your actions and don’t be shy about taking credit for your behavior. What happened? How did the event end? What did you accomplish? What did you learn? Make sure your answer contains multiple positive results.

Make sure that you follow all parts of the STAR method. Be as specific as possible at all times, without rambling or including too much information. Oftentimes students have to be prompted to include their results, so try to include that without being asked. Also, eliminate any examples that do not paint you in a positive light. However, keep in mind that some examples that have a negative result (such as “lost the game”) can highlight your strengths in the face of adversity.

Conceptions

JVM: The Java Virtual Machine is an abstract computing machine. The JVM is a program that looks like a machine to the programs written to execute in it.

JRE:(Java Runtime Environment): It contains JVM program, Java application need to run on JRE.

JDK:(Java Developer Kits): It contains the tools for developing Java programs running on JRE, for example, it provides the compiler “javac”

Java: The source code just need to compiled once and it can run on any machine with JRE installed. (JRE, not the machine need to recognize the compiled code).

C++: The source code need to be compiled to directly the machine instructions on which machine program is running on. The compiled byte code is different on different machines.

By Checking characters:
  1. Platform compatible, write once, compile once, run everywhere on JVM.

    C++: write once, compile everywhere.
  2. Compiles to Java byte code can be recognized by JVM, independent to underline OS.
  3. Interpreter to routines on systems with different machine codes.
  4. Strongly encourage Object-Oriented Programming Paradigm, everything in Java is class/object.
  5. All types (reference types, primitive types) are always pass by value.
  6. Java does not support unsigned integers
  7. Reference (Java) vs Pointers (C++) No pointer arithmetics
  8. No operator overloading. “+”, “-”…..
  9. Classes/Objects are always allocated on the Heap, there is no way to allocate objects on stack.
  10. Single inheritance, multiple inheritance only be done by implementing multiple interfaces.
  11. Garbage Collection. So we do not need to care about memory management. No memory leak.But the momory use is more than necessary.

Compile and Interpreter

  1. Run compile command “javac” to convert Java source code as JVM readable byte code(.java -> .class)
  2. JVM interprets the Java bytecode to pre-compiled routine on machine doce.
  3. Just-in-time(JIT) compilation to compile the bytecode to native instructions.

The current version of Sun HotSpot JVM uses a technique called Just-in-time(JIT) compilation to compile the bytecode to the native instructions understood by the CPU on the fly at run time.
Some implementations of JVM might interpret the bytecode instead of JIT compiling it to machine code and running it directly. While this is still considered an “interpreter”. It is significantly different from interpreters that read and execute the high level source code.

Stack VS. Heap:

Stack: local variable, tracking method call flow, return address etc. – one per thread

Heap: dynamic memory allocation. – only one per JVM

The keyword,“new”, means allocate memory on Heap for the created object, and return the reference of the object.

The JAVA does not allow to relocate memory in stack.

Garbage Collection:

Why do we need it? What exactly the problem it tries to solve?

More efficiently manage dynamic memory allocation. In program, it has to create some “Objects” to solve the problems, and these things are dynamically created and need some memory to store them.

C++:

  • “new” to crate object on heap
  • “delete” to reclaim the allocated memory, and this is needed to be done by developer themselves! (imaging how hard it is..)

Java:

  • Garbage Collection is mechanism to automatically detect/delete the unused objects, so that developers do not need to care too much about each individual object life cycle.

The heap stores object data, and this area is also managed by the garbage collector at startup.

How does GC work:

  1. Marking
  2. Normal Deletion
  3. Deletion with Compacting
    Other: When GC works, the whole program will not work even threats

JVM Generations:

The information learned from object allocation behavior can be used to enhance the performance of the JVM. Therefore, the heap is broken up into smaller parts or generations.

The heap parts are:

  • Young Generation
  • Old or Tenured Generation
  • Permanent Generation.

If GC is moving the objects in heap, that means the main java program should be stopped, this is called “stop the world” event. This happens each time if the GC needs to run, the running time of GC is related to the heap size, if we have a very large heap (heap size can be vary large, for example, 100G is normal case for some application), delay of response will be a big problem here. Hence, the Generations idea is used for the application.

The reason of generations
a. Do not stop the world very long every time.
b. The pattern of the object life cycles could be utilized.

Design Principle:

Most of the objects used in application/program has short life cycle. Having hierarchical generation will make GC more efficient to avoid long stop when doing on the full heap. The generation is defined by the length of life cycle of the objects, and the GC is only running on the generations with longer cycle when needed.

Old Generation:

Typically, a threshold is set for young generation object and when that age is met, the object gets moved to the old generation (the basic escalation rule).

Permanent generation:
metadata required by the JVM to describe the classes and methods used in the application, and the String intern.
Classes may get collected if the JVM finds they are no longer needed and space may be needed for other classes. The permanent generation is included in a full GC.

How to know if an object can be GCed or not?

  • Graph of dependency of refence and objects
  • From a set of root references/objects, find all the objects reachable through the paths on the dependency graph, other ones are not needed and can be GCed.

  • To solve dynamic constructor problem, the nested static class, “XXBuilder” could invoke the private outer class constructor.

Final Keyword

Constant. “Once assigned, cannot be changed.”

  • Final Class: A class that cannot be derived. It has no child (no inheritance).
  • Final method: A method has cannot be overridden. It could not be override.
  • Final field: A variable that once assigned, it cannot be assign again even in constructor. It could not be changed;

Other Keywords:

Classic Types (objects): stored in heap, operated by references (reference object).

‘this’ Keyword: The current(self) instance. Mandatory if argument name is the same as field name.

‘null’ keyword: is special value for the reference. It means that reference is empty.

NullPointerException: It means it makes JAVA dereferences the reference (NULL).

Function

Funtion is also named as “Method”.

  • Access sepcifier: It defines accessibiliy from others.
  • Return type: It defines return type. If there is no value to return, it should be void.
  • Method name: A method Identifier
  • Parameter list: It defines variables from outside. It is optional.
  • Method body: It defines logic for method.

Passing Parameters

In C#, it defines 3 ways to pass parameters to a function.

  • Pass by Value: It is a default way to pass parameters. When a method recieve parameters as “Pass by Value”, it will create new memory to copy and save these parameters. When the copid value is changed, it will not affect the value from outside.

E.g.

  • Pass by Refence: The method will get the memory address of the passed parameters. All changes in this method will be directly affected in memory.
  • Pass Out: It allows method to receive parameters that are not defined and returns parameters back.

Comapare to JAVA

JAVA has no principle for “Pass by Value” and “Pass by Referance” and “Pass Out”.

If a method receive parameters, it will do “Pass by Value” for primitive parameters. Also, it will do “Pass by Value” for non-primitive parameters.

Encapsulation & Inheritance & Polymorphism

Encapsulation:
It is defined as the wrapping up of data under a single unit. It is the meachansim that binds together code and data it manipulates. Other way to think about encapsulation is that is a protective shield that prevents the data from being accessed by the code outside this shield.

  • Technically in encapsulation, the variables or data of a class is hidden from any other class and can be accessed through &any member function of own class in which they are declared.
  • As in encapsulation, the data in a class is hidden from other classes, so it is also known as data-hiding.
  • Encapsulation can be achieved by: Declaring all the variables in the class as private and writing public methods in the class to set and get the values of variables.

Advantage of Encapsulation

  • Data Hiding: The user will have no idea about the inner impelmentation of the class. It will not be visible to the user that how the class is storing values in the varaibles. He only knows that we are passing the values to a setter method or variables are getting initialized with that value.

  • Increased Flexibility: WE can make the variables of the class as read-only or write-only depending on our requriement. if we wish to make the variables as read-only then we have to moit the setter methods. For write-only, the program should omit get methods.

  • Reusability: Encapsulation also improves the re-useability and easy to change with new requiements.

  • Testing code is eassy: It is easy to test for unit testing.

Access modifier

Access modifier defines different way to access a method or a varaible.

  • public: The type of member can be accessed by any other code in the smae assembly or other assembly that references it.
  • private: The type or member can be accessed only by code in the same class or struct
  • protected: The type or memeber can be accessed only by code in the same class, or in a aclass that is drived from that class.
  • internal: the type or member can be accessed by any code in the same assembly, but not from another assemly.
  • protected internal: The type or member can be accessed by any code in the assebmly in which it is declared, or from within a derived class in another assmblely.
  • private protected: The type or memeber can be accessed only within its declaring assemly, by code in the same class or in a type that is derived from that class.
