Java-JVM Memory: Stack and Heap:

Stack: LIFO, a call stack is a stack data structure that stores information about the active subroutines of a computer program. This kind of stack is also known as an execution stack, program stack, control stack, run-time stack, or machine stack, and is often shortened to just “the stack”. It is used to track the instructions.

Heap: Java objects reside in an area called the heap. The heap is created when the JVM (Java Virtual Machine) starts up and may increase or decrease in size while the application runs.

Heap & Stack Comparison:

  • Local primitive type variables are always on stack
  • Whenever an object is created, its always stored in the Heap space and stack memory contains the reference to it.
  • Memory management in stack is done in LIFO manner; memory management in heap is more complex and requires complex algorithm
  • Stack is like the data structure “stack” because both are LIFO, heap has little to do with the data structure “heap”.
  • Stack variable is destroyed after out of scope. Heap variable is destroyed by the garbage collector.
  • (advanced & optional) Each thread has its own stack; multiple threads share the same heap.
  • (advanced & optional) When stack memory is full, Java runtime throws java.lang.StackOverFlowError
    Whereas if heap memory is full, it throws java.lang.OutOfMemoryError: Java Heap Space error.
  • (Advanced & optional) Stack is smaller than heap; storing data to stack is faster than heap.
  • Frequently, Stack is far smaller than Heap. It is easier to have OVER FLOW ERROR.

Not all variables saved in Stack, but all objects are saved in Heap.

Any Static members (fields, methods, classes) belong to class, not object. The member could be accessed by [ClassName].[Static field].

Java does not know how to create an instance, so it is using Main function, “public static void main(String[] args)”.

Advantage, Nested Static Class:

  • The nested static class could be instantiated before the instantiated outer class.
  • To solve dynamic constructor problem, the nested static class, “XXBuilder” could invoke the private outer class constructor.

Constant. “Once assigned, cannot be changed.”

  • Final method: A method has cannot be overridden. It could not be override.
  • Final field: A variable that once assigned, it cannot be assign again even in constructor. It could not be changed;

Encapsulation & Inheritance & Polymorphism

Encapsulation:
It is defined as the wrapping up of data under a single unit. It is the meachansim that binds together code and data it manipulates. Other way to think about encapsulation is that is a protective shield that prevents the data from being accessed by the code outside this shield.

  • Technically in encapsulation, the variables or data of a class is hidden from any other class and can be accessed through & any member function of own class in which they are declared.
  • As in encapsulation, the data in a class is hidden from other classes, so it is also known as data-hiding.
  • Encapsulation can be achieved by: Declaring all the variables in the class as private and writing public methods in the class to set and get the values of variables.

Advantage of Encapsulation

  • Data Hiding: The user will have no idea about the inner impelmentation of the class. It will not be visible to the user that how the class is storing values in the varaibles. He only knows that we are passing the values to a setter method or variables are getting initialized with that value.

  • Increased Flexibility: WE can make the variables of the class as read-only or write-only depending on our requriement. if we wish to make the variables as read-only then we have to moit the setter methods. For write-only, the program should omit get methods.

  • Reusability: Encapsulation also improves the re-useability and easy to change with new requiements.

  • Testing code is easy: It is easy to test for unit testing.

Access modifier

Access modifier defines different way to access a method or a varaible.

  • public: The type of member can be accessed by any other code in the smae assembly or other assembly that references it.
  • private: The type or member can be accessed only by code in the same class or struct
  • protected: The type or memeber can be accessed only by code in the same class, or in a aclass that is drived from that class.
  • internal: the type or member can be accessed by any code in the same assembly, but not from another assemly.
  • protected internal: The type or member can be accessed by any code in the assebmly in which it is declared, or from within a derived class in another assmblely.
  • private protected: The type or memeber can be accessed only within its declaring assemly, by code in the same class or in a type that is derived from that class.

Binary Search

WIKI:

Binary Search is a search algorithm to find a position of a target in a sorted array.

Hints:

  • Applicable

    It is for question that ask array contains target or not.

  • Overflow

    int mid = i + (j - i) / 2; -> avoid overflow

    int mid = (j + i) / 2; -> i + j may be greated then Integer.MAX_VALUE;

  • Dead Lock

    While (i <= j) will check the element even i == j. When i = j - 1, the mid value will be the same as i. Then, i = mid will cause dead lock.

Enhancement Binary Search:

Sometimes, the array may contains mutiple targets, and requirement wants to consider another condition to get the target such as first appearance or last appearance or the most closest value.

Hints:

  • Applicable

    It is applicable for an array that contains mutiple available elements for the input target in a sorted array, and finally the mothod will get at least one element from available elements.

  • Loop Termination

    For these questions, the loop should be stopped if the range only contains 2 elements (It also covers special case that array only contains 1 element). According to different, the loop could be stopped if the range contains n elements.

    Why should loop stop with at least 2 elements? There is no variable to record all available elements durding the loop processing.

What is Coding Interviews

2019-05-01
Algorithm
  • Principle

Interviewer Conditions

Coding Interview is going to select a perfect matched programer to work together. Hence, interviwers will assume that interviewees have already has coding experience and focus on communication skill and working style.

Questions

Normally, interviewers will ask project related questions such as Binary Search Serialization or Deserilization, mutiple threads, timing LRU, and so on. If interviewees are not familier with these questions, interviewers will try to find advatages of these interviewees by asking open questions or hard coding questions.

Communication

Interviewers will check communication skill by ask coding questions. They want interviewees ask some questions before typing code because real tasks are not the same as coding questions. It is easier to cause mistakes by ignoring inconspicuous changes.

You can follow below steps to answer algorithm questions:

Before

  • Ask Questions to document your assumptions. (Limitation, Value Type, Focus on Space or Time)
  • Briefly Explain your approach or solution to show how you intend to solve the problem. (2 or 3 sentences)

When you make some assutions, the interviewers may ask you change your assumptions. They want to see your reaction to meet unknow questions.For example, if you assume time cost is higher priority, they may ask you to assume sapce cost is hither priorty.

Coding

  • Provide code comments where applicable. (the # of parameters of a method are over 3, that should write comment);

Bug Free and complete on time.

After

  • Explain the big-O run time complexity of your solution. Justify your answer.
  • Identify and additional data structures you used and justify why you used them. (Why do you use this structure? Is that necessary?).
  • Only provide your best answer to each part of the question.

Then, you could run some corner cases to check code logic. Next, interviwers will ask follow-up questions in order to check your knowledge and patience.

Data structure

WIKI:

It is a data organization, management and storage format that enables efficient access and modifications.

GG:

It is a particular way of organizing data in a computer so that it can be used efficiently.

Both of these definations refer two key keywords: origanizaiton and efficient. Programmers will use certain methods to originize data to reduce time cost and space cost which are the most important meansures to judge algorithm solutions.

Time Complexity

WIKI
Time complexity describes the amount of time it takes to run an algorithm. Time complexity is commonly estimated by counts the number of elementary operations performed by the algorhm.

E.g.

Normally, time compleixty is increased with increasing the number of input. All algorithm problems consider input as “n”.
Mostly, time compleixty of solving an algorithm problem could be random number in a range such as from n to n^2, programers commonly consider the worst-case time complity such O(n), O(nlogn), O(n^2) and so on.

Theta(n): tight bound
Omega(n): lower bound

Space Complexity

WIKI

The space compleixty of an algorithm or a computer program is the amount of momery space required to solve an instance of the computer problem as a function of the size of the input.

In other words, how much memory does it need to run the problem.

However, this defination is not used to answer algorithm questions during interviews. Because interviwers wants “Auxiliary Space Compleixty”.

Auxiliary Space complexity: how much extra space or temporary space will be used by an algorithm.
